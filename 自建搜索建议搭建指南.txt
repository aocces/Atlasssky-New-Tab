自建 Cloudflare Worker 代理指南
这个项目的搜索建议功能，建议通过你自己的 Cloudflare Worker 代理来请求。这样更稳定，也更适合浏览器扩展使用。

第一步：注册并登录 Cloudflare
1.	先注册一个 Cloudflare 账号并登录后台。
第二步：创建 Worker
1.	打开 Cloudflare 后台
2.	进入 Workers & Pages
3.	点击 Create
4.	选择 Worker
5.	输入一个名字，例如：search-proxy
6.	创建后进入在线编辑器

1.	点击右上角的 Deploy 或 Save and Deploy。部署成功后，你会看到一个地址，类似：https://search-proxy.your-subdomain.workers.dev
第五步：测试 Worker 是否正常
1.	在浏览器里直接打开下面两个地址测试。
	测试 Bing

https://你的-worker.workers.dev/?engine=bing&q=apple

正常时会返回类似：
```json
["apple", ["apple", "apple watch", "apple id"]]
```
	测试 DuckDuckGo
```text
https://你的-worker.workers.dev/?engine=duckduckgo&q=apple
```

正常时会返回一个 JSON 数组。
如果能正常返回 JSON，说明 Worker 已经可用了。

 第七步：重新加载扩展
1.	打开 `chrome://extensions/`
2.	找到你的扩展
3.	点击刷新按钮
4.	回到新标签页测试搜索建议
 常见问题
 1. 打开 Worker 地址提示 400
这通常是你没带参数。
必须带上：

 `engine`
 `q`

例如：

```text
?engine=bing&q=apple
```
 2. Bing 没有返回数据
请确认你用的是这条接口：

```text
https://api.bing.com/osjson.aspx?query=关键词
```

不要再用旧的 JSONP 方式。
 3. DuckDuckGo 没有返回数据
先直接在浏览器打开 Worker 测试地址，确认是 Worker 问题还是扩展接入问题。
 4. 想换成自己的域名
可以。
Cloudflare 官方支持把 Worker 绑定到自定义域名，后续你可以把 `workers.dev` 换成自己的正式域名。后面会出教程，敬请期待。
